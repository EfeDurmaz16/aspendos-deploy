# YULA v1.5 + v2+ Backlog

> **Status**: Deferred from v1 launch scope (15-day sprint locked in `docs/superpowers/specs/2026-04-07-yula-manus-alternative-design.md`). This is the prioritized roadmap for post-launch iteration, informed by deep research into OpenClaw (351k ⭐ MIT) and Hermes Agent (34k ⭐ MIT) codebases on 2026-04-07.
> **Owner**: Efe Baran Durmaz
> **Related**: ADR 0001, v1 design spec, v1 implementation plan

## Context

After v1 launch (day 15), YULA will have: Convex + WorkOS + AI SDK v6 + Next 16 + 8 messaging surfaces + 5 reversibility-class reference tools + `/undo` + web `/timeline` + E2B + Steel + Computer Use + Stripe 3-tier + launch videos. This backlog is what comes next, ranked by customer-persona fit and build cost.

The lessons come from a deep technical + UX investigation of OpenClaw's repo (`src/chat/heartbeat.ts`, `src/channels/plugins/types.plugin.ts`, `src/security/{dangerous-tools,external-content,skill-scanner,audit}.ts`, `docs/concepts/{agent-loop,memory,dreaming}.md`, `AGENTS.md`) and Hermes Agent's repo (`run_agent.py:1694-1764` skill learning loop, `tools/skill_manager_tool.py`, `tools/registry.py`, `gateway/platforms/`, `agent/smart_model_routing.py`, `hermes_state.py`, `scripts/install.sh`).

## v1.5 — First post-launch sprint (target: day 30-60)

### Visible self-improvement (Hermes closed loop, YULA-governed)

**The single most valuable pattern from Hermes research.** Hermes's `_spawn_background_review()` mechanism (`run_agent.py:1694-1764`) is genuinely novel product theater — users *feel* the agent getting better because they can literally `ls ~/.hermes/skills/` and see new files appear. But Hermes has a critical bug (Issue #6051 "learned helplessness") where transient failures become permanent anti-knowledge. YULA's governance overlay fixes exactly this.

| # | Feature | Effort | Value |
|---|---|---|---|
| 1 | **Background skill reviewer** — after every N tool iterations, spawn a cheap-model reviewer agent that asks "was this approach reusable?" and writes a skill file if yes | 3d | High — viral demo |
| 2 | **Approval-gated skill creation** — unlike Hermes (silent writes), YULA posts a card: "I learned how to [X]. Save as reusable skill? [Save] [Skip]" | 1d | Critical — matches trust positioning |
| 3 | **SKILL.md format with YAML frontmatter** — YAML `name/description/version/tags/related_skills` + markdown body, git-able, diff-able, human-editable | 1d | Medium — interop with agentskills.io |
| 4 | **Skill TTL + re-validation** — every 30 days, skill runs against recent data; if no recent hits, marked stale; user notified | 1d | Critical — avoids Hermes learned-helplessness bug |
| 5 | **Confidence scoring** — skills written during failure-context get low scores, auto-delete if <0.3 | 0.5d | Critical — fail-close on bad learning |
| 6 | **Visible `/skills` UI** — web page showing learned skills, when learned, how often used, user can edit/delete | 1d | High — "YULA learning visibly" marketing |
| 7 | **Skills guard scanner** — static analysis on auto-generated skills detects dangerous patterns, prompt-injection payloads (borrowed from OpenClaw `skill-scanner.ts` + Hermes `skills_guard.py`) | 1d | Critical — security |

**Sub-total**: 8.5 days

## v1.5 TOTAL: ~30 days of work

Suggested pacing: 2-3 post-launch sprints over 45-60 calendar days. Prioritize in this order by customer-persona impact:

1. **Sprint 1 (days 30-45)**: Features 1-7 (skill learning loop) + 11-12 (doctor diagnostics) + 17 (export). Ship the visible-improvement story + compliance-dev must-haves.
2. **Sprint 2 (days 45-60)**: Features 8-10 (PERSONA.md + memory export) + 13-14 (weekly digest) + 16 (cost caps/BYOK). Add user portability + engagement loops.
3. **Sprint 3 (days 60-75)**: Features 15 (hook system) + 18 (REST API) + 19 (agentskills.io) + 20 (replay). Power-user wedge + interop.

i18n (22) runs in parallel as background infra, not a sprint.

## Anti-patterns — things to NEVER ship (from research)

From OpenClaw/Hermes mistakes:

1. **❌ Unsandboxed user-generated skill marketplace** — OpenClaw's ClawHub shipped 341 malicious skills ([source](https://thehackernews.com/2026/02/researchers-find-341-malicious-clawhub.html)). Any skill marketplace YULA ships must be curated + FIDES-signed + skills-guard-scanned.

2. **❌ Loopback trust model** — OpenClaw's "localhost is trusted" assumption was exploited (ClawJacked CVE, [source](https://www.oasis.security/blog/openclaw-vulnerability)). YULA's WorkOS auth on every endpoint, including local dev.

3. **❌ Monolithic run_agent.py** — Hermes has a 9,400-line AIAgent class. YULA orchestrator files MUST stay under 300 LOC each, split by responsibility.

4. **❌ LLM-judged skill worthiness on main model** — Hermes runs the full main model on every skill-creation trigger. YULA uses cheap Haiku + deterministic classifier.

5. **❌ Skills with zero TTL or confidence scoring** — Hermes Issue #6051 "learned helplessness". YULA enforces TTL + re-validation + failure-context fail-close.

6. **❌ Claude / Claud / Clawd naming adjacency** — OpenClaw got renamed twice in two months due to Anthropic trademark pressure. YULA + Aspendos are safe. Audit every future sub-brand.

7. **❌ Today+yesterday auto-load memory** — OpenClaw's pattern bloats context linearly. YULA uses sliding window + semantic search.

8. **❌ Per-platform duplicated adapter files** — Hermes `gateway/platforms/` has 15+ near-duplicate files. YULA uses Vercel Chat SDK + slot composition (Pattern 2 in v1 spec).
